# Spice v1.2.0 (Apr 28, 2025)

Spice v1.2.0 introduces parameterized queries, enabling secure and efficient dynamic parameter binding in SQL queries. This release also adds support for workers as configurable units of compute, upgrades to DataFusion 45, Arrow 54, DuckDB 1.2.2, and Rust 1.85, bringing performance improvements and stability.
The FinanceBench benchmark suite is now integrated into CI for AI/LLM performance evaluation.

## Highlights in Spice v1.2.0

- **Parameterized Queries**: Spice now supports parameterized queries via both the Flight SQL API and HTTP API, allowing users to define and execute queries with placeholders for dynamic values.

- **Workers**: Introduced workers as configurable units of compute in the Spice runtime that coordinate and manage interactions between models and tools. Workers are defined as components in the `spicepod.yaml` file, allowing for customized behavior and interaction logic.

Example worker configuration for LLM round-robin routing:

```yaml
workers:
  - name: round-robin
    description: |
      Distributes requests between 'foo' and 'bar' models in a round-robin fashion.
    models:
      - from: foo
      - from: bar
  - name: fallback
    description: |
      Attempts 'bar' first, then 'foo', then 'baz' if previous models fail.
    models:
      - from: foo
        order: 2
      - from: bar
        order: 1
      - from: baz
        order: 3
```

- **SQL Query Plan Caching**: Enhanced the results cache with logical plan caching for SQL statements, significantly improving query performance for repeated parameterized queries.

- **Databricks Model Support**: Added support for using Databricks models with `from: databricks:model_name` syntax.

Example `spicepod.yaml` configuration:

```yaml
models:
  - from: databricks:llama-3_2_1_1b_instruct
    name: llama-instruct
    params:
      databricks_endpoint: dbc-46470731-42e5.cloud.databricks.com
      databricks_token: ${ secrets:SPICE_DATABRICKS_TOKEN }
```


- **FinanceBench Integration**: Integrated the FinanceBench benchmark suite into CI for consistent evaluation of AI/LLM performance on financial data queries.

- **Upgrades**: Upgraded to DataFusion 45, Arrow 54, DuckDB 1.2.2, and Rust 1.85 with Edition 2024, providing better query performance, improved type handling, and increased stability.
