# Spice v1.0.4 (Feb 13, 2024)

Spice v1.0.4 introduces partition pruning for Delta Lake time-based partitions, improving scan efficiency and reducing overhead. xAI tool calling is also more robust and the `spice trace` CLI command now provides expanded, detailed output for deeper analysis. In addition, a bug has been fixed to correctly apply column name case-sensitivity in refresh SQL, indexes, and primary keys.

## Highlights in v1.0.4

- **Partition pruning for Delta Lake**: Partitioned Delta Lake tables will now completely eliminate partitions that are not needed to satisfy the query, dramatically reducing the amount of data loaded and processed for queries that filter on the partition column.

Example:

```bash
my_delta_table/
├── _delta_log/
├── date_col=2023-12-31/
├── date_col=2024-02-04/
├── date_col=2025-01-01/
└── date_col=2030-06-15/
```

The query `SELECT * FROM my_delta_table WHERE date_col = '2025-01-01'` will only read data from the `date_col=2025-01-01` partition.

- **`datasets.time_partition_column`** Using append-based acceleration refreshes with a `time_column` ensures that only newly created records are added to the acceleration. However, the physical partitioning of a dataset will often be a larger bucket of time than what the `time_column` represents. By specifying the `time_partition_column` and `time_partition_format`, Spice will use the partitioning to eliminate unnecessary partitions from the acceleration refresh.

```yaml
datasets:
  - from: delta_lake://my_delta_table
    name: my_delta_table
    time_column: created_at # A fine-grained timestamp
    time_format: timestamp
    time_partition_column: date_col # Data is physically partitioned by `date_col` that could be used to completly eliminate unnecessary partitions
    time_partition_format: date
```

## Upgrading

To upgrade to v1.0.4, use one of the following methods:

**CLI**:

```console
spice upgrade
```

**Homebrew**:

```console
brew upgrade spiceai/spiceai/spice
```

**Docker**:

Pull the spiceai/spiceai:1.0.4 image:

```console
docker pull spiceai/spiceai:1.0.4
```

For available tags, see [DockerHub](https://hub.docker.com/r/spiceai/spiceai/tags).

**Helm**:

```console
helm repo update
helm upgrade spiceai spiceai/spiceai
```
